Java基础笔记

1.static是静态修饰符，可用于修饰变量、方法、内部类等。（类属性和实例属性，静态方法和实例方法。static的作用是把类的成员变成类相关，而不是实例相关，static修饰的成员属于整个类，而不属于单个对象。）

​ 静态变量的定义必须写在方法之外，因为静态变量为这个类公共拥有，不属于某个方法，在未赋初始值时系统会默认为静态变量赋初始值。类属性的生命周期随着类的存在而存在，实例属性的生命周期随着实例的存在而存在。

​ 因为类成员的作用域大于实例成员的作用域，所以静态成员不能直接访问非静态成员，需要用对象。否则可能会由于静态变量先初始化而实例变量未初始化，在访问时发生未定义等大量错误。

​ 使用static修饰的方法属于这个类，或者说属于这个类的所有对象公共拥有，调用方法为”类.方法()”。未使用static修饰的方法为这个类的某个对象(自己new出来的)所拥有，调用方法为”对象.方法()”。

访问控制符 (访问控制级别由小到大)：private -> default -> protected -> public

2.return用于结束一个方法并返回一个基本数据类型或引用类型(对象)，当没有返回值时就直接结束一个方法，也可与continue，break 一样用于循环语句或条件判断语句中。

3.类的复用：继承与组合。

继承是对已有的类改造成适用于某些特定需求的类 (override 覆写 / 重写)。(子类继承父类)

组合是当两个类有明显的整体、部分的关系时，将一个类作为另一个类的属性，嵌入到该类中并实现特定的功能。(嵌入类嵌入被嵌入类)

两者的系统开销本质上并无差别。继承的super关键字调用父类方法时，系统隐式的创建了父类的对象，super关键字引用的就是该对象。组合则是显示的创建了嵌入类的对象。所以总的系统开销并无差别。

4.抽象类(abstract) 和 接口(interface)。抽象类和接口都是从多个子类中抽象出来的共同特征，但抽象类主要作为多个类的模板，而接口则定义了多个类应该遵守的规范。

5.instanceof运算符：obj instanceof class，前面的参数是对象 / 引用类型变量，后面的参数是类 / 接口，如果前面的对象是后面的类、子类、实现类的实例，则返回true。

6.final修饰符：可以修饰变量、方法、类。

final修饰基本数据类型变量只能初始化一次，不可再次赋值（变为常量了）；修饰引用类型变量初始化后，不能改变引用，但可以改变引用的对象的属性。

final修饰的方法不可被重写（继承后子类不可重写父类的final修饰后的方法），但依旧可以重载。

final修饰的类不能被继承，该类不可有子类。

7.== 和 equals 比较运算符。

== 用于基本数据类型时，数值相等返回true；用于引用类型时，指向同一对象返回true。（Java中通过new关键字创造出来的都是新的对象 / 引用类型数据，一个新的引用，但引用的可能是同一个对象）（构造器负责创造新的对象的引用）

equals 只是在判断两个变量的值，用于基本数据类型、引用类型时，都是在判断”值相等”。

8.valueOf() 用于返回给定参数的值（基本数据类型、String的原生值）。

9.抽象类不能用于创建实例，只能被其子类继承。抽象类中的抽象方法必须由子类提供实现(子类重写父类的抽象方法)。

final修饰的类不能被继承，所以final修饰的方法也不能被重写，因此final 和 abstract 不能同时使用。

抽象类没有方法体，没有方法可以调用，static修饰的方法为静态方法可以直接被类调用，所以static 和 abstrcat 也不能同时修饰某个方法。

abstract 修饰的方法必须被其子类重写才有意义，所以abstract 方法不能定义为private访问权限，即private 和 abstract不能同时使用。

10.接口 (interface)。接口里不能包含普通方法，接口里都是抽象方法。接口的好处是让规范和实现分离。

接口–> 类 –> 对象，类是接口的实现，对象是类的实例化。

11.对象在内存中的状态：

  1. String str = new String(“hello”); 与 String str = “hello”;

    JVM有栈内存(stack)、堆内存(heap)，堆内存中有字符串常量池用于存储字符串常量。简单理解：栈中存放对象(引用)，堆中存放数据。
